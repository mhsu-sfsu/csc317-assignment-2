How to Apply CSS to HTML

In order to include an external CSS file in an html file, we have to link the two files together. In the head of the html file in question, we would include the following line:

<link rel="stylesheet" href="/style.css">

Note that the keyword link here is not the same as creating a link to another page as we want the CSS to be applied to this page rather than being redirected to the CSS file.
The rel attribute used here is what lets us link the two files and the href attribute specifies where the file we are linking (in this case, the CSS file) is in relation to the html file.


The resulting html file should look something like this:

<html>

<head>

<link rel="stylesheet" href="/style.css">

<\head>

<body>

<\body>

<\html>

Any styling done to this file can now be done in the CSS file, creating a cleaner and easier coding experience!



How to Use JavaScript in a Webpage


When using JavaScript in a webpage, the more common method is to link an external file similar to the CSS file. Another method that can very helpful in some instances is using JavaScript directly in the HTML file; however, it is important to note that both are used in different ways and have their strengths and weaknesses.
Linking a JavaScript file to an html file is very similar to linking a CSS file. The main difference is that instead of using the attribute link, we instead use the script tag. This tag specifically allows us to embed the script into the html file. The line to include would look like this:

<script src="myscripts.js"></script>

Similarly to the linking of a CSS file, we include the file path relative to the html file we want to link to. Instead of using href we instead use src, which stands for source. This line can be placed in multiple areas, but should generally be placed before it is being used.

The html file including both external CSS and JavaScript files should look something like this:

<html>

<head>

<link rel="stylesheet" href="/style.css">

<\head>

<body>

<script src="myscripts.js"></script>

<\body>

<\html>

JavaScript can now be referenced in this html file from an external source!

If you would rather try another option, it is possible to write JavaScript directly into your html file as well. In order to do so, the script tag is used again. Similarly to linking a file, there will be both two tags. However, there won't be a source asigned to the script because the code will instead be directly in the HTML file. In between the starting and ending tab, any JavaScript can be added at this point.

An example of a simple JavaScript line is the following, which alerts with the words "hello world!" to the page.

alert("hello world!");

If it would be prefered to have this line in the HTML file, then this would be inserted in the space where the script would be used:

<html>

<head>

<\head>

<body>

<script>

alert("hello world!");

<script>

<\body>

<\html>

While there can be advantages to using JavaScript directly inside an HTML file, one of the biggest reasons why this is not the standard is because it creates code that is hard to work with and debug. By using separate files, one is able to easily pinpoint and navigate to areas that may have errors rather than having to look through one larger unorganized file. This is not to say that using it in this way is always a negative, but one should be aware and make decisions about it depending on the situation.
